generation, portfolio exports, and agent-assisted career workflows.

## Profile README Update

Update `Brandon-Gottshall/Brandon-Gottshall` so `About-Me` appears under current
public work:

```md
- [About-Me](https://github.com/Brandon-Gottshall/About-Me) - YAML-driven LaTeX document system for official resume, CV, cover letters, portfolio exports, and agent-assisted tailoring workflows
```

GitHub displays a profile README when the public profile repository matches the
username and contains a root `README.md`.

Source: [Managing your profile README](https://docs.github.com/en/account-and-profile/how-tos/profile-customization/managing-your-profile-readme)

## Pinning

Pin `Brandon-Gottshall/About-Me` and replace `Brandon-Gottshall/dogPark`.

Reasoning:

- `About-Me` is current, practical career infrastructure.
- It demonstrates YAML modeling, LaTeX generation, Python tooling, PDF output,
  portfolio export, and agent workflow design.
- `dogPark` is already framed as a historical hackathon prototype.

GitHub allows up to six pinned repositories or gists.

Source: [Pinning items to your profile](https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/pinning-items-to-your-profile)

## Social Preview

Use `docs/showcase/social-preview.svg` as the design source for a repository
social preview. GitHub recommends PNG, JPG, or GIF under 1 MB, ideally 1280 by
640 pixels, so convert the SVG to PNG before uploading if needed.

Source: [Customizing your repository's social media preview](https://docs.github.com/enterprise-cloud@latest/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/customizing-your-repositorys-social-media-preview)

## Fork Detachment

After this cleanup is pushed, detach from the Awesome CV fork network only after
confirming the permanent tradeoff.

GitHub documents that leaving the fork network is permanent and does not retain
some repository metadata such as issues, pull requests, wikis, stars, watchers,
comments, child forks, and related metadata.

Source: [Detaching a fork](https://docs.github.com/en/enterprise-server@3.20/pull-requests/collaborating-with-pull-requests/working-with-forks/detaching-a-fork)
